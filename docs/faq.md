# 常见问题（已更新至 极简命令版）
本文档汇总了使用 WordFormat 过程中常见的问题及解决方案，帮助您快速排查和解决遇到的问题。

## 命令行使用问题

### Q1：直接输入 `wf` 报错？
**回答**：不会报错！现在直接输入 `wf` 会自动显示**完整使用指南**，不会出现参数错误。

### Q2：命令行执行报参数缺失？
**核心原因**：三个模式必填参数不同
- `gj`（生成JSON）：必须 `-d` `-c`
- `cf`（检查格式）：必须 `-d` `-c` `-f`
- `af`（格式化）：必须 `-d` `-c` `-f`

**正确示例**
```bash
wf gj -d 论文.docx -c 配置.yaml
wf cf -d 论文.docx -c 配置.yaml -f 论文_1234567890.json
wf af -d 论文.docx -c 配置.yaml -f 论文_1234567890.json
```

### Q3：`gj` 模式找不到 `-f` 参数？
**回答**：**`gj` 已经取消 `-f` 参数**！
JSON 会**自动生成**在 `-o` 目录，文件名 = 文档名 + 10位时间戳，无需手动指定。

### Q4：`cf / af` 提示 JSON 文件不存在？
**排查方向**
1. 确认已先执行 `wf gj` 生成 JSON
2. 确认 `-f` 后面是**完整的 JSON 文件路径**，不是文件夹
3. 检查路径是否正确（可直接复制终端打印的 JSON 路径）

### Q5：`wf` 或 `wordformat` 命令未找到？
**排查方向**
1. 确认进入了虚拟环境
2. 执行过 `pip install -e .`
3. 可使用备用命令：`python -m wordformat`

## 文档处理问题

### Q6：JSON 文件生成失败？
**排查方向**
1. Word 文档路径正确、未损坏、可正常打开
2. 配置文件 `.yaml` 路径正确、格式无错误
3. 文档不是空文件
4. 文件权限正常，可读取

### Q7：格式校验/格式化没有效果？
**排查方向**
1. JSON 文件中 `category` 分类正确（标题/正文等）
2. YAML 配置规则书写正确
3. 文档无异常格式（手动换行、分节符、表格嵌套异常）
4. 若文档本身符合规范，则不会生成批注/修改

### Q8：生成后的 Word 文档打不开？
**排查方向**
1. 仅支持 `.docx`，不支持 `.doc`
2. Python 版本 >= 3.11
3. JSON 文件未被手动改坏（语法正确）
4. 输出目录有写入权限

### Q9：文档结构识别不准确？
**解决方法**
1. 标题层级要清晰
2. 可手动打开 JSON 修改 `category` 字段修正
3. 更换适配的配置文件（本专科/硕士/期刊等）

## 环境配置问题

### Q10：依赖安装失败？
**解决方法**
1. Python >= 3.11
2. 更新 pip：`pip install --upgrade pip`
3. 使用 uv 安装：`uv sync`
4. 检查网络

### Q11：API 服务启动失败？
**排查方向**
1. 端口 8000 未被占用
2. 依赖完整安装
3. `.env` 配置正确

## Git 相关问题（保持不变）

### Q12：SSH 推送 Git LFS 卡住/失败
**原因**：GitHub 不支持 SSH 上传 LFS

**解决**
```bash
git remote set-url origin https://github.com/AfishInLake/WordFormat.git
```

### Q13：Git 出现大量变更文件
**解决**：确保 `.gitignore` 忽略 `output/`、`tmp/`、`__pycache__/` 等目录

## 性能问题

### Q14：大文档处理慢
1. 属于正常现象
2. 确保内存 ≥ 8G
3. 关闭其他占用资源程序

### Q15：内存过高崩溃
1. 拆分文档处理
2. 重启程序
3. 清理后台程序
