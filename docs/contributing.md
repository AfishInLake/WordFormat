# 贡献指南

感谢您对 WordFormat 项目的关注和支持！本文档将指导您如何为项目贡献代码、文档和其他资源，共同完善这个论文格式自动化处理工具。

## 贡献方式

### 1. 提交 Issue

您可以通过提交 Issue 来：
- 反馈 bug
- 提出新功能建议
- 补充格式配置模板需求
- 讨论项目发展方向

### 2. 提交 Pull Request

您可以通过提交 Pull Request 来：
- 修复 bug
- 新增格式检查规则
- 完善核心功能
- 优化文档
- 改进代码质量

### 3. 完善配置模板

您可以通过以下方式完善配置模板：
- 补充不同学校/期刊/学位的论文格式配置YAML文件，提交到 `example/` 目录
- 改进现有模板，使其更加准确和全面
- 为模板添加详细注释，提高可读性

## 开发规范

### 1. 代码规范

- 代码遵循 PEP 8 编码规范
- 使用清晰的变量/函数命名，避免使用缩写和模糊命名
- 保持代码风格一致性，包括缩进、空格和换行
- 避免使用魔法数字，使用命名常量代替

### 2. 注释规范

- 新增功能/规则需添加对应的注释
- 关键逻辑添加使用示例
- 复杂算法添加详细说明
- 保持注释与代码同步，避免过时注释

### 3. 提交规范

- 提交信息清晰明了，使用英文或中文
- 提交信息应包含：
  - 变更类型（feat/fix/docs/style/refactor/test/chore）
  - 变更内容的简要描述
  - 相关 Issue 编号（如果有）

### 4. 测试规范

- 新增功能需添加对应的测试用例
- 确保所有测试通过后再提交 Pull Request
- 测试用例应覆盖正常情况和边界情况
- 测试代码应简洁明了，易于理解

## 开发流程

### 1. 环境准备

1. **克隆仓库**
   ```bash
   git clone https://github.com/AfishInLake/WordFormat.git
   cd WordFormat
   ```

2. **创建虚拟环境**
   ```bash
   uv venv
   # 或使用 venv
   python -m venv venv
   ```

3. **激活虚拟环境**
   ```bash
   # Windows
   venv\Scripts\activate
   # macOS/Linux
   source venv/bin/activate
   ```

4. **安装依赖**
   ```bash
   make install
   # 或
   pip install -e .
   ```

### 2. 开发流程

1. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   # 或
   git checkout -b fix/your-bug-fix
   ```

2. **开发功能**
   - 实现功能或修复 bug
   - 添加测试用例
   - 完善文档

3. **运行测试**
   ```bash
   # 运行所有测试
   pytest
   # 运行特定模块的测试
   pytest tests/test_specific_module.py
   ```

4. **检查代码质量**
   ```bash
   # 检查代码风格
   flake8
   # 检查类型注解
   mypy src/
   ```

5. **提交代码**
   ```bash
   git add .
   git commit -m "feat: add new feature"
   git push origin feature/your-feature-name
   ```

6. **创建 Pull Request**
   - 访问 GitHub 仓库
   - 点击 "Pull requests" 选项卡
   - 点击 "New pull request"
   - 选择您的分支和目标分支
   - 填写 Pull Request 描述
   - 点击 "Create pull request"

### 3. 代码审查流程

1. **维护者审查**：项目维护者会审查您的代码变更
2. **反馈修改**：根据审查意见修改代码
3. **测试验证**：确保所有测试通过
4. **合并代码**：审查通过后，代码会被合并到主分支

## 文档贡献

### 1. 文档结构

- `docs/` 目录下的文档按功能和使用场景分类组织
- 文档使用 Markdown 格式编写
- 文档应清晰明了，结构合理

### 2. 文档类型

- **教程文档**：面向新用户的入门指南
- **参考文档**：详细的 API 和配置项说明
- **示例文档**：常见使用场景的示例代码
- **贡献文档**：如何为项目贡献代码和文档

### 3. 文档规范

- 使用清晰的标题层级
- 代码示例使用代码块格式
- 表格使用 Markdown 表格格式
- 图片使用相对路径引用
- 链接使用 Markdown 链接格式

## 配置模板贡献

### 1. 模板结构

- 配置模板存放在 `example/` 目录下
- 模板使用 YAML 格式编写
- 模板应包含详细注释

### 2. 模板类型

- **学位论文模板**：本科、硕士、博士毕业论文模板
- **期刊论文模板**：不同期刊的论文格式模板
- **会议论文模板**：不同会议的论文格式模板

### 3. 模板规范

- 模板名称应清晰明了，包含适用范围
- 模板应包含完整的格式配置项
- 模板应添加详细注释，说明各配置项的含义
- 模板应经过测试验证，确保能够正常工作
